# Governance

## Overview

Wolkvorm is an open source project licensed under the Apache License 2.0. This document describes the governance model for the project.

## Roles

### Users

Anyone who uses Wolkvorm. Users are encouraged to participate by filing issues, providing feedback, and helping other users.

### Contributors

Anyone who contributes code, documentation, schemas, or other improvements via pull requests. All contributions are subject to the project's [Contributing Guidelines](CONTRIBUTING.md).

### Maintainers

Maintainers have commit access and are responsible for the overall direction of the project. Current maintainers are listed in [MAINTAINERS.md](MAINTAINERS.md).

Maintainers are expected to:
- Review pull requests in a timely manner
- Participate in project discussions
- Follow the Code of Conduct
- Act in the best interest of the project and community

## Decision Making

- **Routine decisions** (bug fixes, minor features, documentation) are made by individual maintainers via PR review and merge.
- **Significant decisions** (architecture changes, new cloud providers, breaking changes) require discussion in a GitHub Issue or Discussion and consensus among maintainers.
- **Disputes** are resolved by discussion. If consensus cannot be reached, the Lead Maintainer makes the final decision.

## Adding Maintainers

New maintainers are nominated by existing maintainers based on:
- Sustained, high-quality contributions over at least 3 months
- Demonstrated understanding of the project architecture
- Positive interactions with the community

Nominations require approval from a majority of existing maintainers.

## Removing Maintainers

Maintainers who are inactive for more than 6 months may be moved to emeritus status. Maintainers may also voluntarily step down at any time.

## Changes to Governance

Changes to this governance document require approval from all active maintainers.
