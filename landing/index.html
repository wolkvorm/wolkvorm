🚀 Quick Start

Get Wolkvorm running in under 5 minutes with Docker Compose.

1

Prerequisites

Install Docker and Docker Compose on your machine. You'll also need AWS credentials (IAM role or access key).

2

Clone and Build

git clone https://github.com/wolkvorm/wolkvorm.git
cd wolkvorm
docker compose up -d --build
3

Open in Browser

Navigate to http://localhost:3000. Create your admin account on first visit, then configure your AWS credentials in Settings.

ServicePortDescription
Frontend3000React UI with nginx proxy
Backend8080Go API server

☁️ Deploy to EC2

Run Wolkvorm on a dedicated AWS EC2 instance for team use.

1

Launch an EC2 Instance

  • AMI: Ubuntu 22.04 LTS
  • Type: t3.medium (2 vCPU, 4 GB RAM)
  • Storage: 30 GB gp3
  • Security Group: Open ports 22, 3000, 8080
  • IAM Role: Attach a role with permissions for AWS services you want to manage
2

Install Docker and Run

ssh -i your-key.pem ubuntu@YOUR_IP

# Install Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker ubuntu
newgrp docker

# Clone and run
git clone https://github.com/wolkvorm/wolkvorm.git ~/wolkvorm
cd ~/wolkvorm
docker compose up -d --build
3

Access

Open http://YOUR_IP:3000 after ~3 minutes (initial build).

Push Updates

# From your local machine
rsync -avz --exclude 'node_modules' --exclude '.git' \
  -e "ssh -i your-key.pem" \
  ./wolkvorm/ ubuntu@IP:~/wolkvorm/

ssh -i your-key.pem ubuntu@IP \
  "cd ~/wolkvorm && docker compose up -d --build"

⚙️ Configuration

Configure Wolkvorm through the Settings page in the UI.

AWS Credentials

Go to Settings > AWS:

  • IAM Role (recommended for EC2) — No credentials needed, uses instance profile automatically.
  • Access Key — Enter your AWS Access Key ID and Secret Access Key.
  • Default Region — Set the default region for new resources.

S3 State Backend

Terraform state is stored remotely in S3 for durability and team access.

  1. Go to Settings > AWS and enter an S3 bucket name.
  2. Optionally configure a DynamoDB table for state locking.
  3. Go to My Resources and click Initialize State Backend.

GitHub Integration (optional)

Generate Terraform HCL and open pull requests from the UI.

  1. Create a GitHub App with repository read/write permissions.
  2. Go to Settings > GitHub.
  3. Enter the App ID and upload the private key PEM file.

Cost Estimation (optional)

Go to Settings > Infracost and enter your Infracost API key to enable pre-deploy cost estimates.

🧩 Adding Custom Resources

Every resource in Wolkvorm is defined as a JSON schema file. Add new resources by creating a JSON file in schemas/aws/.

{
  "id": "my-service",
  "name": "My Service",
  "description": "Description of the service",
  "provider": "aws",
  "icon": "my-service",
  "category": "compute",
  "module": {
    "source": "git::https://github.com/...",
    "outputs": ["arn", "id"]
  },
  "inputs": [
    {
      "name": "name",
      "label": "Service Name",
      "type": "string",
      "required": true
    }
  ],
  "common_inputs": {
    "region": true,
    "environment": true,
    "tags": true
  }
}

Input Types

Restart the backend after adding a new schema. The resource will appear automatically in the UI.

📐 Instance Sizing Guide

Choose the right EC2 instance based on your team size and concurrent operations.

UsersInstanceRAMMonthly Cost
1 (dev/test)t3.medium4 GB~$30
2-3 userst3.large8 GB~$60
4+ userst3.xlarge16 GB~$120

Each concurrent terraform plan/apply operation uses approximately 200-500 MB RAM in a Docker container. A t3.medium comfortably handles 2-3 simultaneous operations.

Storage

Minimum 30 GB gp3 recommended. Docker images, Terraform provider cache, and state files consume storage over time. Monitor with df -h and clean with docker system prune periodically.
